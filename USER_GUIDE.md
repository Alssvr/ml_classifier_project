# 📘 Руководство пользователя классификатора

## 👋 Для кого это руководство

Это руководство для **экспертов по продукции** и **менеджеров**, которые хотят:
- Автоматически классифицировать новые товары
- Проверять и исправлять предсказания модели
- Улучшать точность классификации со временем

**Вам не нужно знать Python или программирование!** Все действия выполняются простыми командами.

---

## 🚀 Быстрый старт (5 минут)

### Шаг 1: Подготовьте компьютер (один раз)

1. Установите Python с [python.org](https://python.org)
2. Скачайте проект с GitHub (зелёная кнопка "Code" → "Download ZIP")
3. Распакуйте архив в удобную папку
4. Дважды кликните на `install.bat` — он сам всё настроит

### Шаг 2: Положите данные в нужную папку

Создайте или скопируйте Excel-файлы в папку `data/raw/`:

data/raw/  
├── classified_train.xlsx # Уже размеченные товары  
└── classified_new.xlsx # Новые товары для классификации


**Важно!** Файлы должны иметь колонки:
- `ID` — уникальный номер
- `Наименование` — название товара
- `Шаблон класса` — категория (только в размеченном файле)

### Шаг 3: Запустите классификацию

1. Дважды кликните на `classify_new.bat`
2. Подождите 2-5 минут
3. Готово! Результаты в папке `data/processed/`

---

## 📂 Где что лежит (шпаргалка)

| Папка | Что там | Когда смотреть |
|-------|---------|----------------|
| `data/raw/` | Исходные Excel-файлы | Перед запуском |
| `data/processed/` | Результаты классификации | После запуска |
| `data/feedback/` | Файлы для проверки экспертами | Когда нужно что-то перепроверить |
| `models/` | Обученные модели | Не трогайте 😊 |

---

## 🔄 Три режима работы

### Режим 1: Просто классифицировать новые товары

**Когда использовать:** У вас есть новые товары, нужно быстро получить категории.

**Что делать:**
1. Положите файл `classified_new.xlsx` в `data/raw/`
2. Дважды кликните `classify_new.bat`
3. Заберите результат из `data/processed/classified_[дата].xlsx`

**Что вы получите:**
- Excel-файл с колонками: `ID`, `Наименование`, `predicted_class`, `confidence`
- `confidence` — насколько модель уверена (от 0 до 1, выше 0.1 — хорошо)

---

### Режим 2: Проверить сомнительные предсказания (работа эксперта)

**Когда использовать:** Модель выдала низкую уверенность для некоторых товаров.

**Что делать:**

1. **Найдите файл для проверки**
   - После классификации в `data/processed/` появится `needs_review_[дата].xlsx`
   - Откройте его в Excel

2. **Заполните колонки для каждой строки:**
   
   | Колонка | Что писать | Примеры |
   |---------|------------|---------|
   | `expert_decision` | `CONFIRM` — согласен с моделью<br>`CORRECT` — исправить на другой<br>`SKIP` — пропустить | `CORRECT` |
   | `expert_label` | Правильная категория (если CORRECT) | `Подшипники конические` |
   | `expert_comment` | Любое примечание | `Опечатка в названии` |

   Пример заполнения:
   
   | Наименование | predicted_class | expert_decision | expert_label |
   |--------------|-----------------|-----------------|--------------|
   | Распределитель | Тесто | CORRECT | Распределители |
   | Подшипник 32215 | Подшипники | CONFIRM | |
   | Непонятно что | Разное | SKIP | |

3. **Сохраните файл**
   - Назовите с суффиксом `_reviewed`
   - Например: `needs_review_20260422_reviewed.xlsx`
   - Положите в `data/feedback/`

4. **Запустите дообучение**
   - Дважды кликните `improve_model.bat`
   - Модель обучится с учётом ваших правок

---

### Режим 3: Добавить новые размеченные данные

**Когда использовать:** Появилась новая партия уже проверенных товаров.

**Что делать:**
1. Добавьте новые строки в `data/raw/classified_train.xlsx`
2. Дважды кликните `retrain_model.bat`
3. Модель переобучится на расширенных данных

---

## ⚠️ Важно: как правильно обновить данные для обучения

Если вы **заменили или дополнили** файл `classified_train.xlsx` новыми записями, модель **НЕ обучится** на них автоматически. Нужно принудительно перезапустить обучение.

### 🔄 Полный сброс и переобучение

1. **Закройте все Excel-файлы** из папки `data/`
2. **Дважды кликните** `reset_and_retrain.bat`
3. Подождите 5-10 минут
4. Готово — модель обучена на новых данных

### 📁 Что делает `reset_and_retrain.bat`

- Удаляет старую модель
- Очищает временные файлы
- Запускает обучение с нуля на **текущих** данных из `classified_train.xlsx`

### ❔ Когда нужно переобучать

| Действие | Нужно ли переобучать |
|----------|---------------------|
| Заменили файл `classified_train.xlsx` | ✅ Да |
| Добавили новые строки в `classified_train.xlsx` | ✅ Да |
| Заменили `classified_new.xlsx` | ❌ Нет (просто запустите классификацию) |
| Изменили `config.py` | ✅ Да |

## ❓ Часто задаваемые вопросы

### ❔ "Модель предсказала 'Тесто' для подшипника. Почему?"

Это нормально для первой версии. Отметьте такие случаи как `CORRECT` в режиме 2, и после дообучения модель исправится.

### ❔ "Что означает confidence 0.05?"

Модель совсем не уверена. Такие записи **обязательно** нужно проверять эксперту.

### ❔ "Сколько записей проверять?"

Начните с 200-500 записей с самой низкой уверенностью. Этого достаточно для заметного улучшения модели.

### ❔ "Можно ли использовать модель для других товаров?"

Да, но точность будет низкой. Лучше обучить новую модель на ваших данных (режим 1, но с вашим размеченным файлом).

## 📊 Как понять, что модель стала лучше

После каждого дообучения смотрите в консоль — там будет строка:

Accuracy: 0.8956 (было 0.8942)


Чем ближе к 1.0, тем лучше. Рост на 0.01 — уже хороший результат.

## 🆘 Что делать, если что-то пошло не так

### Проблема: "Файл не найден"

**Решение:** Проверьте, что:
- Файл лежит в правильной папке (`data/raw/`)
- Название точно совпадает: `classified_train.xlsx` или `classified_new.xlsx`

### Проблема: "Программа закрылась с ошибкой"

**Решение:**
1. Сделайте скриншот ошибки
2. Отправьте техническому специалисту
3. Не паникуйте — данные не пострадали

### Проблема: "Не могу открыть Excel после обработки"

**Решение:** Закройте Excel перед запуском программы. Программа не может сохранить файл, если он открыт.
