Сильные стороны ML

✅ Замкнутая терминология — TF-IDF отлично работает
✅ Скорость критична для ERP-систем
✅ Предсказуемость и детерминизм
✅ Прозрачность решений
✅ Экономическая эффективность (дешевле в 10-20 раз)

Слабые стороны ML

❌ Обработка OOV терминов (RFID-метка XC-890)
❌ Контекстное понимание сложных конструкций
❌ Медленная адаптация к новым классам
❌ Плохое качество на редких классах (5 примеров)

Сильные стороны LLM

✅ Zero-shot обучение
✅ Понимание сложных конструкций
✅ Работа с синонимами
✅ Актуальность новых терминов
✅ Объяснение решений

Слабые стороны LLM

⚠️ Галлюцинации (выдуманный класс)
⚠️ Высокая стоимость ($0.01 за запрос → $300/мес)
⚠️ Латентность 2-5 секунд
⚠️ Нестабильность результатов
⚠️ Зависимость от API провайдера

🎯 Финальный вердикт

Для КЛАССИФИКАЦИИ технических наименований:

  • ✅ Базовое решение: ML (TF-IDF + LR) — оставьте как основной
  • 🚀 Усиление: Гибридный роутер + LLM как fallback
  • 🌀 Долгосрочно: Active Learning + LLM обратная связь для генерации данных

ML + SBERT + локальный LLM на сложных случаях = ИДЕАЛ
